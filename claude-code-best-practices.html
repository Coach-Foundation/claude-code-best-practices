Claude Code Best Practices

Save tokens, reduce costs, and get better results

This guide covers the most impactful ways to make Claude Code faster, cheaper, and more effective. These tips come from official Anthropic documentation, community testing, and real-world benchmarks. No coding knowledge required.

The Big Picture: Every message you send to Claude Code carries your entire conversation history plus configuration files. Small inefficiencies multiply across hundreds of messages. The changes below can reduce your usage by 50-70%.

1. Settings That Save Tokens Automatically

These are one-time changes to your settings.json file. Once set, they work silently in the background.

Thinking Token Cap High Impact

Claude "thinks" before responding, and those thinking tokens cost 5x more than regular tokens. Capping them at 10,000 (from the default of ~32,000) saves roughly 70% on thinking costs with minimal quality loss.

Cheaper Subagents High Impact

When Claude spawns helper agents (subagents) to work in parallel, they default to using the same expensive model. Routing subagents to Haiku (a faster, cheaper model) saves ~80% on subagent costs. Subagents mostly read files and return summaries - they don't need the full reasoning power.

Effort Level Medium Impact

The "effort level" controls how deeply Claude reasons about each response. Setting it to "medium" (instead of "high") reduces output tokens by 50-70%. You can always type ultrathink in a message when you need deep reasoning for a hard problem.

Earlier Auto-Compaction Medium Impact

When your conversation gets too long, Claude automatically compresses it. By default this happens at ~95% capacity. Setting it to 75% triggers compaction much earlier, giving more buffer room and preventing emergency compactions that can lose important context.

How to apply these settings

Add this to your ~/.claude/settings.json (or ask Claude Code to do it for you):

"env": { "MAX_THINKING_TOKENS": "10000", "CLAUDE_CODE_SUBAGENT_MODEL": "haiku", "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "75" }, "effortLevel": "medium"

2. Keep Your CLAUDE.md Small

Your CLAUDE.md file is loaded into every single message you send. A large file means thousands of extra tokens on every turn.

The Rule of Thumb: Keep CLAUDE.md under 100 lines. Think of it as an email with links, not an employee handbook.
DoDon't
Short, specific rules Claude can't guess Long explanations of standard practices
Move specialized workflows into Skills Cram everything into one file
Link to detailed docs with @path/to/doc.md Paste entire API docs inline
Rules that prevent real mistakes "Write clean, readable code" (Claude does this by default)
The Test High Impact

For every line in your CLAUDE.md, ask: "Would Claude make a mistake without this?" If the answer is no, delete it. Each unnecessary line costs you tokens on every single message.

3. File Exclusions (.claudeignore)

Add exclusions to your project's .gitignore file, or create a .claudeignore in your project root. This prevents Claude from accidentally reading or indexing large, irrelevant files. (Claude respects .gitignore by default - a dedicated .claudeignore is optional for additional exclusions.)

# Add these to .gitignore or .claudeignore node_modules/ dist/ build/ .next/ *.lock __pycache__/ .git/ *.db *.sqlite *.log coverage/ *.min.js *.min.css vendor/

One user reported that just adding .next/ cut their context usage by 30-40% in a Next.js project.

4. Session Habits That Save Tokens

These are behavior changes - no configuration needed, just awareness.

Keep Sessions Short High Impact

Token cost accelerates as conversations grow. By message 30, a single prompt costs ~167,000 tokens because the entire history is re-sent. Start fresh sessions for new tasks. A series of short sessions is dramatically cheaper than one long marathon.

Never Resume Old Sessions High Impact

Resuming a day-old conversation forces Claude to re-process the entire history at full price (the cache has expired). Multiple users reported losing 20-28% of their usage window on a single resumed message. Start a new session instead.

Mind the 5-Minute Cache Medium Impact

After 5 minutes of inactivity, the prompt cache expires. Your next message costs roughly 10x more because everything gets reprocessed. If you step away for coffee, your return message is expensive. Subscription users get a 1-hour cache, which helps significantly.

Use /clear Between Tasks Medium Impact

Type /clear when switching to an unrelated task. Accumulated context from the previous task wastes tokens on every subsequent message. A clean slate is almost always cheaper.

Use /btw for Side Questions Medium Impact

The /btw command asks a question in a popup that never enters your conversation history. Perfect for quick questions like "what does this error mean?" without bloating your context. Can reduce token usage by up to 50% in question-heavy sessions.

After 2 Failed Fixes, Start Fresh Medium Impact

If Claude is going in circles trying to fix something, type /clear and write a better prompt that includes what you learned from the failed attempts. A clean session with a good prompt beats a bloated session with accumulated confusion.

5. Writing Better Prompts

How you phrase requests directly affects how many tokens Claude uses to respond.

Instead of this...Try this...
"How does the auth system work?" "In src/auth/middleware.ts, how does token refresh work?"
"Fix the bug" "The login form crashes when email is empty. Error: [paste error]. Fix in src/components/Login.tsx"
"Improve this codebase" "Add input validation to the signup function in api/users.js"
"Run all the tests" "Run tests for the auth module only"
Be Specific, Save Tokens

Vague prompts trigger broad codebase scanning - Claude reads dozens of files trying to understand what you mean. Specific prompts with file paths let Claude work with 1-2 files instead of 20. That difference compounds across every subsequent message.

Paste Errors, Don't Describe Them

Copy-paste the actual error message, stack trace, or CI output. Your description loses details Claude needs and costs extra tokens for less information.

6. Model Selection

Claude Code can use different models for different tasks. Matching the model to the task saves significant tokens.

ModelBest ForRelative Cost
Haiku File lookups, search, renaming, formatting, repetitive tasks 1x (cheapest)
Sonnet Daily coding, tests, simple edits, code generation ~4x
Opus Complex architecture, subtle debugging, multi-file refactors ~15x

Type /model sonnet or /model opus to switch mid-session. Default to Sonnet for everyday work and switch to Opus only when you genuinely need deep reasoning.

7. Useful Commands Quick Reference

Commands You Should Know

  • /clear - Wipe conversation history and start fresh
  • /btw - Ask a side question without adding to history
  • /context - See how your token budget is being used
  • /cost - See API costs for the current session
  • /model [name] - Switch to a different model mid-session
  • /effort [level] - Adjust reasoning depth (low/medium/high/max)
  • /compact [focus] - Compress context, optionally keeping specific topics
  • Escape - Stop Claude mid-action and redirect
  • Escape, Escape - Undo the last action entirely

Recommended Plugins

Type these inside a Claude Code session to install:

  • /install-plugin superpowers - Structured workflows (planning, debugging, code review)
  • /install-plugin context7 - Always-current library documentation
  • /install-plugin code-simplifier - Code quality reviews

8. What NOT to Do

Don't resume old sessions

The cache expires after inactivity. Resuming reprocesses everything at full price.

Don't load every MCP server

Each MCP server adds token overhead on every message. Only enable what you actually need. Check with /mcp.

Don't edit CLAUDE.md mid-session

Changing CLAUDE.md invalidates the prompt cache, making your next message much more expensive.

Don't use Opus for everything

Opus costs ~15x more per token than Haiku. Most coding tasks work perfectly fine on Sonnet.

Don't keep iterating in a broken session

After 2-3 failed attempts at the same fix, /clear and start over with a better prompt. It's cheaper and usually more effective.
