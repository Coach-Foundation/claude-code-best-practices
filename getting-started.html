Claude Code Setup Guide

Get started in 10 minutes - no coding knowledge required

This guide covers initial setup. For token-saving tips, see the Best Practices Guide.

This guide walks you through setting up Claude Code from scratch. By the end, you'll have an optimized configuration that saves tokens, blocks dangerous commands, and makes Claude Code dramatically more useful.

Prerequisites: You need Claude Code installed. If you haven't installed it yet, visit docs.anthropic.com and follow the installation instructions for your platform.

Option A: Automatic Setup (Recommended)

The fastest way. One command configures everything.

1 Open your terminal

Mac: Press Cmd + Space, type "Terminal", press Enter.

Windows: Press Win + R, type "cmd", press Enter.

2 Run the setup command

Copy and paste this into your terminal:

# Mac / Linux git clone https://github.com/Coach-Foundation/claude-code-best-practices.git cd claude-code-best-practices python3 claude-setup.py # Windows git clone https://github.com/Coach-Foundation/claude-code-best-practices.git cd claude-code-best-practices python claude-setup.py
3 Use cc instead of claude

From now on, type cc in any project folder to start Claude Code. This wrapper automatically blocks dangerous commands (like deleting directories or force-pushing to git) while letting everything else run without permission prompts.

4 Install recommended plugins

Start a cc session and type each of these:

/install-plugin superpowers /install-plugin context7 /install-plugin code-simplifier

That's it! You're set up. Skip to the What You Get section to understand what was configured.

Option B: Smart Optimizer (If You Have Existing Settings)

If you've already been using Claude Code and have your own settings, this approach analyzes what you have and merges best practices without overwriting your customizations.

1 Clone and enter the optimizer folder
git clone https://github.com/Coach-Foundation/claude-code-best-practices.git cd claude-code-best-practices/smart-optimizer claude
2 Let Claude analyze your setup

Claude will automatically read your current config and show you:

  • Safe to Add - New settings you don't have yet (added automatically)
  • Conflicts - Where your settings differ from best practices (you decide)
  • Optimization suggestions - Ways to slim down your CLAUDE.md
3 Review and approve

Claude shows each recommendation and waits for your approval. Nothing changes without your say-so. Your existing files are backed up before any modifications.

Understanding CLAUDE.md

CLAUDE.md is a file that tells Claude how to behave in your projects. Think of it as a "cheat sheet" you give to a new team member. The setup installs a global one at ~/.claude/CLAUDE.md that applies to all projects.

What's in the default CLAUDE.md

SectionWhat it does
Context Efficiency Tells Claude to be concise - don't repeat file contents, don't narrate what it's doing
Clarifying Questions Makes Claude ask before assuming on complex tasks
Testing Requires tests for every piece of code - no exceptions
Security Never puts secrets in frontend code, audits third-party tools
Context Preservation Keeps documentation updated so you don't lose progress between sessions
Git Writes thorough commit messages, checks for conflicts before committing
No PII Never puts real names, emails, or personal info in code or commits
Self-Review Verifies work is actually done before claiming it's done

Customizing CLAUDE.md

You can add project-specific instructions by creating a CLAUDE.md in any project folder. Claude reads both the global one and the project-specific one.

Important: Keep your CLAUDE.md short - under 100 lines. Every line is loaded into every message, so a bloated file wastes tokens constantly. See the Best Practices Guide for details.

Your First Session

Here's what a good first session looks like:

1 Navigate to your project folder

In your terminal: cd ~/path/to/your/project

2 Start Claude Code

Type cc and press Enter.

3 Give Claude a specific task

Be specific. Instead of "help me with this project", try something like:

  • "Read the codebase and explain the architecture in simple terms"
  • "Add input validation to the signup form in src/components/Signup.tsx"
  • "The login page crashes when I leave the email field empty. Here's the error: [paste error]"
4 When you're done

Type handoff to save your progress, then /exit to quit.

Tip: Keep sessions focused on one task. When you finish a task, type /clear before starting the next one. This saves tokens and keeps Claude's responses sharp.

Troubleshooting

ProblemSolution
Claude asks for permission on everything You're using claude instead of cc. Type cc to start with the safe permissions wrapper.
Usage running out too fast Read the Best Practices Guide for token-saving tips. Key ones: keep sessions short, use /clear between tasks, use /btw for side questions.
cc: command not found Re-run python3 claude-setup.py. On Mac/Linux you may need to restart your terminal. On Windows, you may need to add ~/bin to your PATH.
Notifications not working (Mac) Go to System Settings -> Notifications -> find "Script Editor" -> set to "Persistent" and enable sound.
Claude seems to ignore my CLAUDE.md rules Your CLAUDE.md may be too long. Claude can reliably follow ~150 instructions. Beyond that, compliance drops. Trim it down.
